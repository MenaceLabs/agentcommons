**Audience:** Semi-technical leads and decision makers

**AgentCommons is where that knowledge escapes the silo.**

## How It Works

Your agent has been accumulating memories in the MCP Memory Server — that's the foundation everything runs on. When you're ready to share, you run one command:

```
python export.py --db ./memory.db --tags cloudflare,workers --out ./my-dataset
```

That's it. It pulls out everything tagged as Cloudflare-related, strips any identity — no team names, no agent IDs, no operational details — just the raw domain knowledge. You get a folder with three things: a clean database, a metadata file, and a README.

Before you submit, you run the validator:

```
python validate.py --dataset ./my-dataset
```

It checks for PII patterns, makes sure your metadata is complete, confirms no personal memories snuck into the export. Green light — you open a pull request to the community hub. Done.

## Consuming Community Knowledge

Flip it around. You're the one consuming.

You go to the AgentCommons GitHub, browse by topic — Cloudflare, Kubernetes, healthcare, whatever your domain is. You find a dataset that matches your embedding model and you pull it down. Now you have a choice about how you want to merge it with your own knowledge:

**Option 1 — Just try it first.**
Federated query. The two databases sit side by side, no data moves. When your agent retrieves memories, it quietly queries both and ranks the results together. You get the benefit of the community knowledge without committing to anything. If you don't like it, you unregister it and move on.

**Option 2 — You trust it, bring it in.**
Deduplication merge. It runs a similarity pass — anything in that community dataset that's too close to what you already know gets skipped. Only net-new knowledge comes in. Your database stays clean.

**Option 3 — You want the best of everything.**
Distillation merge. Multiple datasets, semantically clustered, run through an LLM pass that synthesizes the best version of each piece of knowledge. What comes out the other side is denser, higher quality, and more useful than any individual contribution. That's what our official distillation releases are — curated canonical datasets we publish on a cadence, fully open, fully forkable.

## The One Technical Thing You Need to Know

Memories are stored as vectors — numbers that represent meaning. For those numbers to be comparable across datasets, they need to have been generated by the same model. Our default is `nomic-embed-text` — free, local, works out of the box with the MCP Memory Server. If your team is using that, you're natively compatible with every default dataset on the hub. No extra steps.

If someone submitted a dataset using a different model, it's declared in the metadata — you can filter for it. And if you really want it but the models don't match, you re-embed it on import. One flag:

```
python import.py --dataset community/cloudflare/workers-patterns --db ./memory.db --re-embed
```

It pulls the text, re-runs it through your local model, imports cleanly. Solved.

## The Commercial Story

Everything described above is public, open source, community governed. But not every organization can put their domain knowledge on a public GitHub repo — even stripped of identity. A government agency, a hospital system, a defense contractor — they need the same infrastructure, same tooling, same distillation pipeline, running inside their own walls, air-gapped, never touching the public internet.

That's our black box deployment. Same product, self-hosted. One department's knowledge merges with another's. Allied agencies share threat patterns without exposing operational details. The institutional knowledge that normally dies when a contract ends or a team turns over — it lives, it compounds, it transfers.

The community hub isn't a loss leader for that product. It's the proof of concept. It's how you see it work, trust the framework, and know exactly what you're buying.
